# 考研英语二词汇故事阅读站

这是一个可本地直接打开的静态网站，用来把“50 词故事记忆”和“同词组阅读理解”结合起来。

目前已完成全量 `114` 篇分组内容，可通过左侧目录分页逐篇打开。

## 固定词库规则

这个项目不能随机抓词，必须依赖本地固定词库。

- 原始总词表文件：`source/kaoyan_2025_5500_source.txt`
- 重点词前置名单：`priority_words.txt`
- 重点词生成脚本：`build_priority_words.js`
- 重点词生成说明：`priority_words_meta.json`
- 自动重排脚本：`build_vocab_index.js`
- 当前页面切片：`vocabulary_index.js`
- 篇目归档文件：`story_archive.json`
- 当前 Story 001 固定占用：`001-050`
- 下一篇 Story 002 应固定占用：`051-100`

这样做的目的：

- 保证每次 50 词按编号顺序推进
- 保证不重复
- 保证不遗漏
- 保证后续可以长期归档维护

## 重点词如何确定

你给的 `5500` 文件本身只是完整大纲词表，并不自带“重点词”标签。

所以项目里现在采用两层规则：

1. `source/kaoyan_2025_5500_source.txt` 作为唯一完整来源，不改原始内容。
2. `source/kaoyan2_core_2371.txt` 作为英语二核心高频候选来源。
3. `source/netem_full_list.json` 作为考研词频排序参考。
4. `build_priority_words.js` 会自动生成 `priority_words.txt`。

生成顺序是：

- 先取“英语二核心词表”中和你的 `5500` 总词表重合的词
- 去掉明显不适合做重点词故事起步的功能词、连接词
- 再按“考研词频排序表”的词频顺序排序
- 生成出的 `priority_words.txt` 就是本地 priority 主名单
- 之后再把剩余词按原始总词表顺序依次接在后面

这样“重点词放前、其他顺序在后”就是可复现的，不靠临时判断。

## 使用方式

1. 进入项目目录。
2. 直接双击 `index.html`，或在浏览器中打开它。
3. 在左侧目录中翻页选择篇目。
4. 先在“词汇学习区”阅读中文故事中的目标词。
5. 再到“阅读理解区”完成独立短文的 5 道选择题。
6. 点击“提交答案”查看得分、正确项和解析。

## 在 iPad 上打开

推荐使用局域网访问：

1. 在电脑上双击 `start_ipad_server.bat`
2. 保持弹出的窗口不要关闭
3. 让 iPad 和电脑连接同一个 Wi-Fi
4. 在 iPad 浏览器中打开：

`http://你的电脑局域网IP:8000`

例如：

`http://192.168.1.23:8000`

## 数据维护

所有篇目统一维护在 `stories.js` 中。

每篇数据结构包含：

- `id`
- `title`
- `range`
- `vocabulary`
- `vocabStory`
- `reading`

其中：

- `vocabulary` 是 50 个目标词数组
- `vocabStory` 是词汇学习区内容，包含英文短文和中文对照
- `reading` 是阅读理解区内容，包含英文短文、中文对照和 5 道题目
- 文中使用 `[[word]]` 标记目标词，页面会自动高亮显示

其中词汇顺序必须与 `vocabulary_index.js` 中对应编号区间完全一致。
